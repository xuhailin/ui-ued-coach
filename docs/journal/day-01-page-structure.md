# Day 1 - 页面不是长相，是结构

## Goal

- Learn to judge a page by goal, structure, and attention flow before looking at visual style.
- Build the habit of identifying the first-screen primary action and information hierarchy.

## Core Notes

- The first task of page design is to organize attention, not to decorate.
- Hierarchy is about attention order, not a list of modules.
- A good first screen usually answers three questions quickly: what it is, why it matters, and what to do next.

## Exercise

Page: `https://seko.sensetime.com/explore`

## User Submission

Core goal: 让用户快速体验吧。

First-screen key elements:
1. 然后看到了输入框，快速开始
2. 第一眼看到了立马生成的最佳的效果
3. 包含公告，侧边一些快捷操作。然后都不是第一视觉。滑动有参考的案例。

Hierarchy judgment:
- Level 1: 视觉冲击+快速体验操作。
- Level 2: 加重用户体验，滑动可以看到案例。但具体操作不会出现在这里，重点是首次体验
- Level 3: 快捷操作，侧边。。

Conclusion: 主要是用成品吸引用户，然后让用户快速体验，表明不暴露太多复杂的功能。

## Revised Submission

Core goal: 用高质量生成效果吸引用户，并推动用户立刻开始一次 AI 视频生成尝试。

First-screen key elements:
1. 生成结果/成品效果展示
2. 输入框或快速开始入口
3. 辅助内容，如案例滑动、公告、侧边快捷操作

Hierarchy judgment:
- Level 1: 成品效果展示 + 快速开始入口
- Level 2: 帮助用户建立期待感的案例或辅助说明
- Level 3: 公告、侧边快捷操作、低频辅助信息

Conclusion: 这个页面先用结果建立吸引力，再把用户推向低门槛体验入口，没有在第一屏暴露过多复杂功能，结构上是在服务“先试再说”。

## Coach Feedback

What is already correct:

- The page likely uses output quality to attract attention before explaining details.
- The product is pushing the user toward a quick trial instead of exposing a complex workflow.

What needs revision:

- `Core goal` is still too soft. It should be written as a firm statement, not a guess.
- `Hierarchy` is partly mixing attention order with module listing.

Suggested rewrite:

Core goal:

- 用高质量生成效果吸引用户，并推动用户立刻开始一次 AI 视频生成尝试。

Hierarchy judgment:

- Level 1: 成品效果展示 + 快速开始入口
- Level 2: 帮助用户建立期待感的案例或辅助说明
- Level 3: 公告、侧边快捷操作、低频辅助信息
