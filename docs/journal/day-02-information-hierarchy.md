# Day 2 - 信息层级：用户先看到什么

## Goal

- Learn to judge whether a page is guiding attention in the right order.
- Practice distinguishing primary, secondary, and low-priority information on one screen.

## Core Notes

- Information hierarchy is the order in which the interface wants the user to notice things.
- Strong hierarchy does not mean "more elements"; it means "clear priority".
- If everything is loud, then nothing is loud.

## Exercise

Pick one product page or console page and write:

1. What should the user notice first?
2. What should the user understand second?
3. What can wait until later?
4. Which element is currently too loud or too weak?

## Coach Feedback

What is already correct:

- You correctly noticed that the page leads with product positioning before feature detail.
- You also picked up the key structural fact: clarity comes from placement and order, not only from visual style.

What needs tightening:

- The answer is still mixing observations together; it should be split into "first", "second", and "later".
- The exercise specifically asks which element is too loud or too weak; that part is still missing.
- "有哪些大概操作" is still a bit vague. Try to name the type of understanding more precisely.

Suggested rewrite direction:

- First: product positioning and core value
- Second: supporting proof such as product visuals, examples, or concise explanation
- Later: navigation details, secondary links, and deeper promotional sections
- Too loud / too weak: identify one real element, not a category in general

## Review Questions

1. Can I tell what the first visual focus is within 3 seconds?
2. Is the page trying to make the user think about too many things at once?
3. Is there any secondary information visually competing with the main action?
4. If I remove one element, does the page become clearer?
