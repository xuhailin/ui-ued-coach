# Day 3 - 页面拆解：从整页到模块

## Goal

- Learn to break a full page into functional modules instead of seeing it as one flat screen.
- Practice identifying which modules are primary, supportive, and reusable.

## Core Notes

- A page is not one thing; it is a stack of modules serving one goal.
- Good decomposition separates "structure" from "style".
- If a page cannot be split into clear modules, it is usually hard to design and hard to build.

## Exercise

Pick one page and write:

1. The page goal in one sentence
2. The main modules from top to bottom
3. Which module is primary
4. Which modules are support
5. Which module could become a reusable component pattern

## Review Questions

1. Can I split the page without talking about color or style?
2. Which module carries the main task?
3. Which modules are repeated patterns rather than one-off visuals?
4. If I remove one module, does the page still work?
