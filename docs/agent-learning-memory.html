从 Agent 怎么学，到不同产品场景该怎么记。

这个模块不是技术科普，也不是仓库清单。它把 Agent 学习、记忆类型、产品定位、长对话上下文生命周期和参考实现放在同一个学习控制台里，方便快速扫视和反复查阅。

Q · 01

Agent 学习至少包括哪些能力？

7 类能力，从短期上下文适应逐步走向长期演化。

Q · 02

记忆在学习体系里处于什么位置？

记忆是关键基础设施之一，但 Agent 学习不等于记忆。

Q · 03

不同场景需要什么记忆？

助手型、陪伴型、伴侣型共享基础设施，但边界完全不同。

Q · 04

长对话什么时候压缩、召回、写入？

回答前召回，窗口快满或阶段切换时压缩，回答后抽取可保存的记忆候选。

§ 01

Agent 学习总览

Agent 学习可以拆成 7 类能力。它们不是并列孤岛，而是从当前对话中的短期适应，逐步走向长期经验、策略和行为风格的演化。

学习能力说明典型产物适合场景阶段
上下文学习in_context在当前会话中理解任务、约束、语气、文件、工具状态。当前对话上下文、临时任务状态。所有 Agent短期
用户偏好学习preference记住用户喜欢什么、不喜欢什么、常用表达、工作习惯。用户画像、偏好标签、写作风格。助手型 陪伴型中长
任务流程学习workflow记住用户反复执行的流程、步骤、项目规则。Workflow、SOP、项目上下文。办公助手 开发助手长期
事实与关系学习facts_relations记住人物、项目、事件、组织、地点、对象之间的关系。知识图谱、实体关系、时间线。陪伴型 企业 Agent长期
经验反思学习reflection从失败、纠正、反馈中总结下次怎么做得更好。经验片段、反思规则、失败案例。高阶助手 自动化 Agent演化
行为风格学习persona学习如何表达、如何回应、边界在哪里、什么时候主动。Persona、沟通策略、主动性规则。陪伴型 伴侣型演化
自我改进学习self_improve根据长期反馈调整工具使用、提示词、策略或子 Agent 分工。Prompt 优化、技能选择、策略配置。平台型 高自主 Agent演化
Agent 学习 umbrella
上下文学习 长期记忆 工具使用改进 策略与流程优化 多 Agent 协作经验
长期记忆继续拆为 用户记忆任务记忆关系记忆情绪互动反思经验程序性记忆
§ 02

记忆深度展开

在 Agent 产品里，记忆不是简单保存聊天记录，而是让 Agent 在未来交互中更懂用户、更稳定、更少重复、更能延续关系或任务。

内容事实、偏好、事件、经验或规则。
类型偏好、事实、关系、任务、情绪、反思、禁忌等。
来源来自哪次对话、文件、用户确认、系统事件。
时间发生时间、记录时间、更新时间、失效时间。
置信度用户明确说的，还是 Agent 推断的。
可见性用户是否能看到、编辑、删除。
使用条件什么情况下应该召回，什么情况下不该用。
敏感级别是否涉及隐私、关系、身份、健康、财务等。

§ 03

三类场景：助手 / 陪伴 / 伴侣

这三类是产品定位，不是技术栈。它们可以共享记忆基础设施，但记忆重点、交互边界和风险完全不同。

对比项助手型陪伴型伴侣型
核心目标完成任务长期陪跑亲密关系体验
用户关系工具信任熟悉与理解情感依恋
主动性低到中中中到高
记忆粒度偏任务和规则偏用户状态和事件偏关系细节
情绪权重低中高
风险等级中中高高
§ 04

记忆实现方式

分三层看：怎么存、怎么写、怎么召回。首版不需要全部上，但选型逻辑要先想清楚。

// LONG CONTEXT

长对话：什么时候压缩、召回、写入

一轮对话的上下文生命周期
01 · 当前窗口 保留最近对话 用户刚说的约束、正在改的文件、当前任务状态，优先留在 active context。
02 · 回答前召回 查相关记忆 按项目、实体、意图、时间窗口检索，只注入当前问题需要的少量记忆。
03 · 生成回答 组装 prompt pack 系统规则 + 最近对话 + 滚动摘要 + 检索记忆 + 工具状态。
04 · 触发压缩 生成滚动摘要 当 token 接近阈值、话题切换、阶段完成、工具链很长时，把旧对话压成 summary。
05 · 回答后写入 抽取记忆候选 只从明确偏好、项目规则、重要事件、用户纠正、工具失败经验里抽取。
06 · 入库与下轮 确认后保存 普通项目事实可自动保存；身份、情绪、关系类必须让用户确认，再进入长期记忆。
Context Window 最近 N 轮原文，负责“此刻不断片”。
Rolling Summary 旧对话压缩摘要，负责“长线不丢主线”。
Memory Store 结构化长期记忆，负责“跨会话可召回”。
什么时候召回？ 回答前。先判断当前问题需要项目规则、用户偏好、近期事件还是安全边界，再检索少量相关项。
什么时候压缩？ 窗口快满、阶段结束、话题切换、工具执行链过长时。压缩的是旧上下文，不是高价值长期记忆。
什么时候写入？ 回答后或后台批处理。用户明确说“记住”、纠正 Agent、确认项目规则、工具失败复盘时才写。
什么时候不写？ 普通寒暄、一次性任务细节、不确定推断、敏感情绪和关系信息未确认时，都不要自动进长期记忆。
方式说明优点缺点适合
原始聊天记录保存完整对话，需要时检索或摘要。实现简单，证据完整。噪声大、成本高、隐私压力大。审计、回放、轻量 MVP。
摘要记忆将对话压缩成阶段摘要。成本低，读起来清楚。容易丢细节和来源。陪伴日志、阶段复盘。
结构化字段用 profile、preference、project 等字段保存。可控、可编辑、好展示。灵活性较弱。首版记忆面板
向量记忆用 embedding 检索相关片段。语义召回方便。可能召回错、难解释。通用助手、知识问答。
关键词 / BM25基于关键词检索。可解释、成本低。语义能力弱。名称、项目、明确事实。
混合检索向量 + 关键词 + 重排。召回更稳。系统复杂度更高。中长期产品。
知识图谱建实体、关系、时间与来源。适合关系和时间推理。建模和维护成本高。长期陪伴、企业知识。
时间线记忆按日期、事件、阶段组织。适合回顾和陪伴感。不适合复杂语义检索。陪伴型、成长型。
反思 / 经验库存失败、纠正和策略。能让 Agent 越用越顺。需要质量控制。高阶助手。
用户显式保存用户说“记住这个”或手动添加。适合高价值偏好和敏感信息。
Agent 自动抽取每轮对话后抽取可能有用的记忆。适合普通偏好、项目事实。
后台批处理整理定时总结、去重、合并、过期。适合长期陪伴和知识库。
工具事件写入工具执行成功或失败后记录经验。适合工作流 Agent。
用户确认后写入Agent 提议保存，用户点确认。适合情绪、身份、关系类记忆。
首版建议：核心偏好使用显式保存 + 提议保存；项目记忆可自动抽取但必须可编辑；情绪和关系类记忆必须确认后写入；反思记忆标注“推断 / 总结”。
固定注入每次都把少量核心记忆放进系统上下文。
语义搜索根据用户当前问题检索相关记忆。
时间窗口召回最近一段时间事件，适合陪伴感和阶段复盘。
实体关系查询根据人、项目、对象查询关系，适合图谱型记忆。
规则触发特定任务或页面触发特定记忆。
Agent 自主搜索Agent 判断何时查记忆，适合高级助手。
首版不要让所有记忆默认进入上下文。固定注入少量核心偏好，项目 / 事件记忆用语义搜索 + 时间窗口。
