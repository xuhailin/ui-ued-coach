MemGAS

MemGAS 是一套面向长对话 Agent 记忆检索的论文复现与 quickstart 代码：它把会话、轮次、摘要、关键词等多粒度记忆表示连成关联图，再用路由权重和 Personalized PageRank 做记忆选择。

值得研究算法思路，谨慎直接产品化采用置信度：高

执行摘要

它是什么

这是 ICLR 2026 论文 MemGAS 的开源实现，仓库同时包含实验复现代码和一个 quickstart/ 目录。quickstart 封装了 add/retrieve/save/load 形式的基础记忆 API，原始 src/ 目录则覆盖多粒度信息生成、embedding、关联图构建、检索、生成和评估。

适合谁

适合正在研究长对话记忆召回、会话级/轮次级/摘要级/关键词级多粒度检索、或者想比较 Mem0/Letta/Graphiti 之外算法路线的工程与研究团队。

为什么值得看

它把“记忆该按什么粒度召回”这个问题拆得很清楚：先为同一会话构造多种语义视图，再用图结构表达记忆之间的关联，最后通过 query 相关的重启概率传播选择记忆。这个思路对 xiaoqing 这类长期陪伴/助手型 Agent 很有参考价值。

什么时候不适合

如果目标是开箱即用的生产级 memory server、权限系统、可视化管理、用户可编辑记忆或完整 API 平台，MemGAS 不是直接答案。仓库更像研究代码加轻量 API 封装，缺少测试、许可证、服务层和安全治理。

事实记忆

仓库身份

README.md 将项目命名为 MemGAS，并声明论文 From Single to Multi-Granularity: Toward Long-Term Memory Association and Selection of Conversational Agents 已被 ICLR 2026 接收。

主要语言

仓库主体是 Python 代码，关键目录包括 quickstart/、src/construct/、src/retrieval/、src/generation/、src/evaluation/ 和 data/。

运行形态

不是独立服务或 Web 应用，而是研究复现脚本加 Python API 封装。README.md 提供 conda 环境、pip requirements、数据处理、检索、生成和评估命令。

quickstart API

quickstart/memory.py 暴露 MemGASMemory，支持 add(session, conversation_id, metadata)、retrieve(query, topk, conversation_id, mode)、save() 和 load()。

记忆表示

quickstart/store.py 固定保存 session、turn、summary、keyword、hybrid 五类 embedding；quickstart/schemas.py 保存 session 原文、summary、keywords、metadata 和时间戳。

检索算法

quickstart/retriever.py 中 MultiGranRetriever 支持 session_level、turn_level、summary_level、keyword_level、hybrid_level、lite 和 memgas 模式；memgas 模式会构造多粒度节点图并执行 personalized_pagerank。

多粒度生成

quickstart/llm_client.py 调用 OpenAI-compatible chat completion 生成 summary 和 keywords；src/construct/multigran_generation.py 则为复现实验生成 summary_level 和 keyword_level 日志。

依赖信号

requirements.txt 包含 torch、transformers、sentence-transformers、openai、vllm、python-igraph、scikit-learn、rank-bm25、rouge、evaluate 等研究/评估依赖。

数据来源

README.md 要求下载 LongMemEval、LoCoMo-10、Long-MT-Bench+，data/README.md 指向 HuggingFace 数据集 derongxuaa/MemGAS_data。

质量与治理

仓库未发现 LICENSE、tests/、CI workflow 或安全文档；存在 quickstart/__pycache__ 被提交，以及复现代码中需要手动填 API key/base_url 的位置。

核心设计哲学

  • 把长对话记忆拆成多种粒度，而不是只保存一条摘要或一组向量。

    代码同时维护 session、turn、summary、keyword、hybrid 表示，并允许按单粒度、混合粒度或 MemGAS 图传播方式检索。

  • 检索不是单纯相似度排序，而是通过关联图扩散候选记忆。

    MemGAS 模式将每条记忆的多个粒度展开为节点，通过相似度和 GMM 选择边，再使用 personalized_pagerank 汇总回会话级分数。

    quickstart/retriever.py 和 src/retrieval/run_retrieval.py 都使用基于 softmax 熵的权重逻辑：某一粒度对当前 query 越有区分度，权重越高。

架构

flowchart TD
  User[Python caller / reproduction scripts] --> API[quickstart.MemGASMemory]
  API --> Add[add session]
  Add --> LLM[OpenAI-compatible LLM\nsummary + keywords]
  Add --> Embed[TextEmbedder\nContriever / SBERT]
  LLM --> Store[MemoryStore\nrecords + embeddings]
  Embed --> Store
  User --> Query[retrieve query]
  Query --> QEmb[query embedding]
  QEmb --> Router[entropy router\nper granularity weights]
  Store --> Nodes[multi-granularity nodes\nsession / turn / summary / keyword]
  Router --> Graph[association graph\nGMM edge selection]
  Nodes --> Graph
  Graph --> PPR[personalized PageRank]
  PPR --> Hits[ranked memory hits]
  Data[LongMemEval / LoCoMo / Long-MT-Bench+] --> Scripts[src construct / retrieval / generation / evaluation]
  Scripts --> Logs[process_data / embeddings / graph_cache / retrieval_logs / generation_logs]

MemGAS 可以理解为两层：quickstart/ 提供最小可用记忆 API，src/ 提供论文复现实验管线。核心流程是：会话写入后由 LLM 生成摘要与关键词，embedder 生成多粒度向量，store 持久化 record 与 embedding；检索时 query embedding 进入多粒度路由和图传播，最终返回会话级 memory hits。复现管线则把数据处理、摘要/关键词生成、embedding、图构建、检索、生成和评估拆成脚本。

亮点

  • 多粒度记忆模型非常贴近长对话真实问题

    用户问题有时需要原始会话，有时需要单轮细节，有时摘要足够，有时关键词最能命中实体。MemGAS 把这些视图同时保留下来，而不是押注单一表示。

采用建议

设置复杂度

中到高。quickstart 试用需要 Python 3.9、PyTorch、transformers/sentence-transformers、OpenAI-compatible LLM 或 vLLM；完整复现还要下载外部数据集并准备 GPU 环境。

集成路径

建议先只用 quickstart/ 做小样本验证：构造几段用户长期对话，写入 MemGASMemory，比较 session_level、summary_level、keyword_level、lite 和 memgas 模式的召回差异。

扩展点

可扩展 embedder、LLM provider、MemoryRecord 字段、router 权重策略、GMM 选边参数、PageRank 参数，以及是否把图缓存改成持久数据库。

学习曲线

理解 quickstart API 不难；理解 MemGAS 算法需要读多粒度 embedding、熵路由、GMM 边选择和 personalized PageRank。完整复现对环境和数据要求更高。

建议的 PoC

选择 20-50 条模拟 xiaoqing 对话，人工标注 10 个需要跨会话记忆的问题；分别用 summary、keyword、hybrid、memgas 检索 top-3；记录命中率、误召回、可解释性和延迟。

对 xiaoqing 的取舍

V1 不建议直接采用完整 MemGAS；V2 可先做多视图索引；V3 再考虑实体/时间/关系图和图传播。敏感记忆必须先加用户确认和删除机制。
